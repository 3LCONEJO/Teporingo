cómo se arma la red

La red tiene dos brazos que ven cosas distintas pero producen el mismo tipo de salida: un embedding. Luego esos embeddings se comparan como dos piezas que pasan por el mismo calibre.

brazo célula

Entrada: 2F Linear(1024) → BatchNorm1d → ReLU → Dropout Linear(512) → BatchNorm1d → ReLU → Dropout Salida: embedding_dim

brazo genotipo

Entrada: F Linear(512) → BatchNorm1d → ReLU → Dropout Linear(512) → BatchNorm1d → ReLU → Dropout Salida: embedding_dim

comparación

F.normalize(cell_emb) F.normalize(geno_emb) similarity = cosine_similarity(cell_emb, geno_emb)

entrenamiento y ajuste

Se aprende a dar un score alto cuando el par es correcto y uno bajo cuando es incorrecto. El aprendizaje usa batches para estabilizar el gradiente y optimizar más rápido.

parámetros clave

  • loss: SmoothL1Loss
  • optimizer: AdamW
  • learning rate: 1e-3
  • weight decay: 1e-4
  • batch size: 64
  • scheduler: ReduceLROnPlateau
  • early stopping: patience 10
El umbral de decisión no siempre debe ser 0.5. En este proyecto conviene reportar también el mejor threshold y la balanced accuracy.

salida final

El sistema produce un score por par, una tabla de pares, métricas globales y, al final, la asignación de cada célula al donor que mejor encaja.

reportes útiles

  • Accuracy, AUC y AP
  • Brier y log loss para calibración
  • matriz de confusión
  • best_threshold y balanced accuracy
  • top-1 por célula
